Playwright Cheat Sheet (Basic → Advanced)

Topic-wise reference for building a scalable Playwright test framework: locators, fixtures, CI diagnostics, network, auth, and reliability patterns.

Golden rule: prefer Locator + web-first expect, keep selectors stable, isolate data per test, and always capture traces on CI.
Setup & config Locators Assertions & waits Fixtures POM & flows Network mocking Authentication Downloads/uploads Tracing/screenshots/video Parallelism & sharding CI/CD patterns Flake control

1) Setup & config

// playwright.config.ts
import { defineConfig } from '@playwright/test';

export default defineConfig({
  timeout: 60_000,
  expect: { timeout: 10_000 },
  retries: process.env.CI ? 2 : 0,
  workers: process.env.CI ? 4 : undefined,
  reporter: [['html'], ['junit', { outputFile: 'results/junit.xml' }]],
  use: {
    baseURL: process.env.BASE_URL ?? 'https://example.com',
    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
    video: 'retain-on-failure',
  },
});
  • Pin Playwright version (and browsers) for reproducible CI.
  • Use baseURL + relative navigation in tests.

2) Locators (best practice)

  • Prefer getByRole for accessibility and stability.
  • Use data-testid and getByTestId when needed.
  • Avoid brittle CSS/XPath. Avoid text selectors that change often.
await page.getByRole('button', { name: 'Save' }).click();
await page.getByTestId('email').fill('a@b.com');

3) Assertions & waits

  • Prefer web-first assertions (auto-retry) over manual reads.
  • Avoid waitForTimeout. Wait for conditions that represent readiness.
await expect(page.getByTestId('toast')).toContainText('Saved');
await expect(page).toHaveURL(/dashboard/);

4) Fixtures (framework architecture)

// fixtures.ts
import { test as base } from '@playwright/test';

export const test = base.extend({
  adminPage: async ({ page }, use) => {
    // login once per test (or per worker using storageState)
    await use(page);
  },
});

Use fixtures to inject authenticated pages, API clients, and data builders.

5) POM & flows

  • Keep page objects thin: actions + key assertions.
  • Move cross-page flows to “flow” helpers (checkout flow, onboarding flow).
class CheckoutPage {
  constructor(private page) {}
  payNow = this.page.getByRole('button', { name: 'Pay now' });
  async submit() { await this.payNow.click(); }
}

6) Network mocking

await page.route('**/api/user', async route => {
  await route.fulfill({
    status: 200,
    contentType: 'application/json',
    body: JSON.stringify({ name: 'Alice' }),
  });
});

Mock carefully: mock unstable/3rd party dependencies; keep core integrations real in at least one pipeline.

7) Authentication

  • Prefer storageState for fast auth reuse.
  • Create state in global setup or per-worker fixture.
// global-setup.ts
// login once and save storageState.json
// then in config: use: { storageState: 'storageState.json' }

8) Downloads & uploads

const [download] = await Promise.all([
  page.waitForEvent('download'),
  page.getByText('Export').click(),
]);
await download.saveAs('out.csv');

9) Tracing, screenshots, video

  • Use trace: 'on-first-retry' for fast triage.
  • Keep artifacts for failures; clean for passes.
// config
use: { trace: 'on-first-retry', screenshot: 'only-on-failure', video: 'retain-on-failure' }

10) Parallelism & sharding

# shard across CI nodes
npx playwright test --shard=1/4
  • Parallel-safe tests require isolated users/data and no shared files.

11) CI/CD patterns

  • Run smoke on PR, full regression nightly.
  • Upload HTML report + traces as artifacts.
  • Pin Playwright docker image or browsers for consistency.

12) Flake control (senior-level)

  • Selectors: enforce getByRole/getByTestId.
  • Waits: rely on expect retries; avoid time-based waits.
  • Data: create per test; cleanup idempotently.
  • Diagnostics: always keep trace for failures.
// temporary retries with tracking
retries: process.env.CI ? 2 : 0
