CI/CD Automation: GitHub Actions + Playwright

📅 February 20, 2026 📂 DevOps ⏱️ 9 min read

Automated tests are only valuable if they run consistently. GitHub Actions makes it simple to run Playwright tests on every push, pull request, and schedule. This guide shows you how.

Why CI/CD Matters for Automation Testing

  • Catch regressions before code is merged
  • Run tests on multiple OS/browser combinations
  • Generate test reports automatically
  • Fail fast on code quality issues
  • Maintain confidence as the codebase grows

GitHub Actions Setup

Create a workflow file .github/workflows/tests.yml:

name: Playwright Tests

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Install Playwright
        run: npx playwright install
      
      - name: Run Playwright tests
        run: npm run test:e2e
      
      - name: Upload test reports
        if: always()
        uses: actions/upload-artifact@v3
        with:
          name: playwright-reports
          path: playwright-report/

Multi-Browser Testing

Test across Chromium, Firefox, and WebKit:

name: Multi-Browser Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        browser: [chromium, firefox, webkit]
    
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - run: npm ci && npx playwright install
      
      - name: Run tests on ${{ matrix.browser }}
        run: npx playwright test --project=${{ matrix.browser }}
      
      - name: Upload reports
        if: failure()
        uses: actions/upload-artifact@v3
        with:
          name: test-results-${{ matrix.browser }}
          path: test-results/

Test Reporting

Generate HTML reports with test history:

npm test --reporter=html

View detailed reports, screenshots, and videos of failed tests.

Slack Notifications

Notify your team on test failures:

- name: Notify Slack on Failure
  if: failure()
  uses: 8398a7/action-slack@v3
  with:
    status: ${{ job.status }}
    text: 'Tests failed on ${{ github.repository }}'
    webhook_url: ${{ secrets.SLACK_WEBHOOK }}

Performance Monitoring

Track test execution time to catch slowdowns:

- name: Store test metrics
  run: |
    echo "Test duration: $(cat test-results/metrics.json)"
    
- name: Fail if tests too slow
  run: |
    duration=$(jq '.totalDuration' test-results/metrics.json)
    if [ $duration -gt 300000 ]; then
      echo "Tests took too long: ${duration}ms"
      exit 1
    fi

Scheduled Tests (Nightly Runs)

Run comprehensive tests on a schedule:

on:
  schedule:
    - cron: '0 2 * * *'  # Run at 2 AM daily

Best Practices

  • Use npm ci instead of npm install for CI environments
  • Cache dependencies to speed up runs
  • Run tests in parallel when possible
  • Generate HTML reports for every run
  • Upload videos of failed tests for debugging
  • Set reasonable timeouts to prevent hanging
  • Use branch protection rules to require passing tests
