Playwright Interview Questions

Top 15 Playwright interview questions with detailed answers. For freshers to senior QA engineers.

15 Questions Freshers to Advanced Free

15 interview questions — click to expand answers

Q1 What is Playwright and how does it differ from Selenium? +
Playwright is a modern browser automation framework by Microsoft. Key differences: Playwright has auto-waiting built-in (no explicit waits needed for most cases), supports Chromium/Firefox/WebKit natively, runs tests in parallel by default, has better network interception, and supports modern web features like shadow DOM, iframes, and multiple tabs/windows better than Selenium.
Q2 What is a Locator in Playwright? +
A Locator represents a way to find elements on the page. Unlike Selenium's WebElement, Locators are lazy — they don't search for elements immediately. Playwright retries finding the element before each action. Best practice: use getByRole, getByLabel, getByTestId over CSS/XPath selectors.
Q3 What is auto-waiting in Playwright? +
Playwright automatically waits for elements to be actionable before performing actions. For click(), it waits for the element to be visible, stable, enabled, and not obscured. For fill(), it waits for the element to be visible and enabled. This eliminates most explicit waits needed in Selenium.
Q4 How do you handle authentication in Playwright? +
Use storageState to save and reuse authentication: await page.context().storageState({path: 'auth.json'}). In playwright.config.ts, set storageState: 'auth.json' to reuse across tests. Or use HTTP credentials: browser.newContext({httpCredentials: {username, password}})
Q5 What is the difference between page.click() and locator.click()? +
page.click('#btn') is a shorthand that finds and clicks in one step. locator.click() uses a Locator object which is more resilient — it retries finding the element. Prefer locator-based approach: await page.locator('#btn').click() or await page.getByRole('button').click()
Q6 How do you run tests in parallel in Playwright? +
Playwright runs test files in parallel by default. Configure workers in playwright.config.ts: workers: 4. For tests within a file, use test.describe.parallel(). For serial execution: test.describe.serial(). Each worker gets its own browser context.
Q7 What are Playwright Fixtures? +
Fixtures are reusable setup/teardown code. Built-in fixtures: page, browser, context, request. Custom fixtures: extend the test object. Example: const test = base.test.extend({loggedInPage: async ({page}, use) => { await loginAs(page); await use(page); }})
Q8 How do you intercept and mock API calls? +
Use page.route(): await page.route('**/api/users', route => route.fulfill({status: 200, body: JSON.stringify(mockData)})); To abort requests: route.abort(). To modify: route.continue({headers: {...}}). Use page.waitForResponse() to wait for specific API calls.
Q9 What is Playwright Trace Viewer? +
Trace Viewer is a debugging tool that records test execution including screenshots, DOM snapshots, network logs, and console messages. Enable with trace: 'on-first-retry' in config. View with: npx playwright show-trace trace.zip. Essential for debugging CI failures.
Q10 How do you handle multiple tabs/windows? +
Use page events: const [newPage] = await Promise.all([context.waitForEvent('page'), page.click('#openNewTab')]); Then work with newPage as normal. Use context.pages() to get all open pages. BrowserContext manages multiple pages.
Q11 What is the difference between expect() soft assertions? +
Regular expect(): test stops at first failure. Soft assertions: await expect.soft(locator).toBeVisible() — test continues even if assertion fails, collecting all failures. Use expect.soft() when you want to check multiple things and see all failures at once.
Q12 How do you handle file uploads and downloads? +
Upload: await page.setInputFiles('#fileInput', '/path/to/file.pdf'). Multiple files: setInputFiles('#input', ['file1.pdf', 'file2.pdf']). Downloads: const [download] = await Promise.all([page.waitForEvent('download'), page.click('#downloadBtn')]); await download.saveAs('/path/to/save');
Q13 What is Page Object Model in Playwright? +
POM is a design pattern where each page has a class with locators and methods. Example: class LoginPage { constructor(page) { this.page = page; this.username = page.getByLabel('Username'); } async login(user, pwd) { await this.username.fill(user); }}. Makes tests maintainable.
Q14 How do you test responsive design with Playwright? +
Use devices from playwright/test: import { devices } from '@playwright/test'. Configure in playwright.config.ts: projects: [{ name: 'mobile', use: {...devices['iPhone 12']} }]. Or set viewport: await page.setViewportSize({width: 375, height: 812});
Q15 What is the Playwright UI Mode? +
UI Mode (npx playwright test --ui) is an interactive test runner with time-travel debugging, live test execution, filtering, and watch mode. It shows all tests, allows picking individual tests to run, and shows traces inline. Best for local development.
