Robot Framework Interview Questions

Top 20 Robot Framework interview questions with detailed answers. For freshers to experienced QA engineers.

Q1 What is Robot Framework? +
Robot Framework is an open-source, keyword-driven test automation framework written in Python. It supports web, API, mobile, and desktop testing through external libraries like SeleniumLibrary, RequestsLibrary, and AppiumLibrary. Tests are written in a readable tabular format.
Q2 What are the 4 main sections in a Robot Framework file? +
1. *** Settings *** — Library imports, resource files, suite/test setup/teardown
2. *** Variables *** — Scalar (${var}), List (@{var}), Dictionary (&{var})
3. *** Test Cases *** — Individual test scenarios
4. *** Keywords *** — Reusable keyword definitions
Q3 What is the difference between ${}, @{}, and &{}? +
${} is a Scalar variable (single value). @{} is a List variable (ordered collection). &{} is a Dictionary variable (key-value pairs). Example: ${name}=John, @{colors}=red green blue, &{user}=name=John age=30
Q4 What is SeleniumLibrary vs Browser Library? +
SeleniumLibrary uses Selenium WebDriver under the hood (older, stable). Browser Library uses Playwright (newer, faster, auto-waiting built-in). Browser Library supports Chromium, Firefox, and WebKit natively and has better parallelization.
Q5 How do you handle dynamic elements in Robot Framework? +
Use Wait Until Element Is Visible, Wait Until Page Contains, or Wait Until Element Is Enabled keywords instead of Sleep. For XPath, use contains() or starts-with() for dynamic attributes. Example: xpath=//div[contains(@id,'dynamic')]
Q6 What is the difference between Suite Setup and Test Setup? +
Suite Setup runs once before all tests in the suite. Test Setup runs before every individual test. Similarly, Suite Teardown runs once after all tests, and Test Teardown runs after each test.
Q7 How do you run specific tests using tags? +
Use --include and --exclude CLI flags. Example: robot --include smoke tests/ runs only tests tagged with 'smoke'. Multiple tags: robot --include smoke AND login tests/. Exclude: robot --exclude slow tests/
Q8 What is a Resource file in Robot Framework? +
A Resource file (.robot) contains reusable Keywords and Variables shared across multiple test suites. Import with 'Resource path/to/file.robot' in Settings section. It cannot contain Test Cases.
Q9 How do you implement data-driven testing in RF? +
Use test templates with FOR loops, or the DataDriver library for CSV/Excel driven tests. Example: use 'Test Template' keyword in Settings and provide multiple data rows in the Test Cases section.
Q10 What is the difference between Run Keyword If and IF block? +
Run Keyword If is an older approach (RF 3.x style). IF/ELSE IF/ELSE/END is the modern approach (RF 4+). The IF block is more readable and supports nested conditions. Always prefer IF blocks in new code.
Q11 How do you take screenshots on test failure? +
Add in Test Teardown: Run Keyword If Test Failed Take Screenshot (SeleniumLibrary) or Capture Page Screenshot. Or use: Test Teardown Run Keywords Log Source AND Take Screenshot
Q12 What is pabot and when would you use it? +
pabot (Parallel Robot Framework Executor) runs Robot Framework tests in parallel across multiple processes. Use it when test suite execution time is too long. Install: pip install robotframework-pabot. Run: pabot tests/
Q13 How do you handle alerts/popups in RF? +
Use SeleniumLibrary keywords: Handle Alert (accept), Dismiss Alert, Input Text Into Alert. For custom JS dialogs, use Execute Javascript to suppress them or Handle Alert action=ACCEPT
Q14 What is the difference between Click Element and Click Button? +
Click Element works on any HTML element (div, span, a, etc). Click Button works specifically on button elements and input[type=button/submit]. Click Link is for anchor tags. Use Click Element for custom UI components.
Q15 How do you read data from Excel/CSV in Robot Framework? +
Use the DataDriver library for CSV: pip install robotframework-datadriver. For Excel: use openpyxl via Evaluate keyword or create a custom library. Example: ${data} Read CSV testdata.csv
Q16 What is the Set Global Variable keyword? +
Set Global Variable sets a variable that's accessible across all test suites during the run. Set Suite Variable is accessible within the current suite. Set Test Variable is only accessible in the current test.
Q17 How do you handle iframes in Robot Framework? +
Use Select Frame id=iframeId to switch into iframe. Use Unselect Frame to switch back to main content. Use Select Frame xpath=//iframe[@title='myFrame'] for more specific selection.
Q18 What is the purpose of __init__.robot file? +
The __init__.robot file is a suite initialization file. It contains Suite Setup/Teardown that applies to all suites in the directory. It can also contain default settings for all suites in that directory.
Q19 How do you connect to a database in Robot Framework? +
Use DatabaseLibrary: pip install robotframework-databaselibrary. Then: Connect To Database psycopg2 mydb user pwd localhost. Use Execute SQL String, Query keywords to interact with DB.
Q20 What are common Robot Framework best practices? +
1. Use Page Object Model pattern
2. Never use Sleep — use explicit waits
3. Tag all tests for easy filtering
4. Externalize test data
5. Take screenshots on failure
6. Keep keywords small and focused
7. Use meaningful keyword names
8. Version control your tests
